# AI 开发规范

## 技术路线
- 桌面运行时使用 Tauri 2，原生能力由 Rust command 或 Tauri 插件提供。
- 前端使用 Vue 3、Pinia、Vite、SCSS。
- 不继续扩展 Electron；历史 Electron 代码只作为迁移参考。
- 当前 MVP 聚焦本地个人任务管理，不实现账号、云同步、协作、完整日历、习惯、番茄、四象限。

## 工作流程
- 开始编码前先阅读相关代码、文档和当前 git 状态。
- 不回滚用户或其他工具产生的无关变更。
- 每完成一个稳定功能并验证通过后提交一次，commit 使用中文。
- 提交粒度按功能划分，避免把文档、运行时迁移、业务功能修复混在一个提交里。
- 代码变更后至少运行 `npm run build`；涉及 Tauri 时还要运行 `npm run tauri build` 或说明无法运行的原因。

## 代码约定
- Vue 组件保持组合式 API 写法。
- Pinia store 只负责任务状态、派生数据和业务操作；平台文件读写通过 `src/services/platform.js` 统一封装。
- Rust command 只处理本地原生能力，例如应用数据目录读写、文件选择、图片读取。
- 数据模型必须向后兼容旧 JSON，新增字段提供默认值。
- UI 保持滴答清单风格：低饱和青绿色、紧凑信息密度、三栏/四栏桌面布局、操作按钮按需显现。

## 文档约定
- 产品决策写入 `docs/产品需求文档.md`。
- 架构和接口写入 `docs/技术架构.md`。
- 数据结构写入 `docs/数据模型.md`。
- 竞品和功能范围写入 `docs/产品调研.md`、`docs/功能路线图.md`。
