# 帮助中心与截图维护说明

## 内容边界

应用内帮助的唯一内容源是 `src/components/HelpCenter.vue`。它面向使用者，覆盖快速开始、任务与子任务、富文本备注、视图、设置、个人空间、数据维护、快捷键和更新说明。

当功能变化时，按以下顺序维护：

1. 先确认界面和行为已经真实可用。
2. 更新 `HelpCenter.vue` 的文字、关键词、入口描述和截图说明。
3. 如对外可见，再更新 README、[安装与使用](安装与使用.md) 或产品需求文档。
4. 最后更新对应截图；不可访问的入口、规划功能或过期截图不得留在帮助中心。

数据维护入口固定为“个人空间 → 空间管理”；恢复点入口固定为“个人空间 → 数据与安全”。

## 截图规范

- 统一桌面视口：`1280 × 800`，WebP 格式。
- 截图须来自稳定演示数据，文案、标签和状态能说明功能；不得出现真实个人数据、浏览器边框或其他应用内容。
- 未展示任务详情时关闭详情面板；展示属性、子任务或备注时必须有明确选中任务。
- 截图必须反映当前 UI。更改设置入口、布局、平台提示或功能状态后，优先检查相关图片。
- 图片放在 `public/screenshots/`，引用时提供准确 `alt` 与说明。

## 生成流程

演示数据为根目录 `demo-data.json`。先更新日期或场景，再运行：

```bash
node scripts/seed-demo-data.mjs
npm run frontend:dev -- --host 127.0.0.1
SIMPLE_TODO_SCREENSHOT_URL='http://127.0.0.1:5173/' node scripts/capture-manual-screenshots.mjs
```

截图脚本会生成并覆盖 `today-view`、`task-detail`、`subtasks`、`rich-editor`、`planned-view`、`important-view`、`inbox-view`、`search`、设置、个人空间和帮助中心等图片。脚本使用浏览器环境的 localStorage 演示回退，只用于截图，不代表桌面版的正式数据路径。

## 发布前检查

- 帮助中心、README、安装说明和 Release Notes 对当前版本的能力表述一致。
- 所有图片文件存在，名称与代码引用一致；必要时人工检查画面与说明是否相符。
- `npm run frontend:build` 通过后再提交帮助文案或截图变更。
