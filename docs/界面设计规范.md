# 界面设计规范

## 布局
- 桌面端采用四栏布局：窄图标栏、清单侧栏、任务列表、任务详情。
- 窄图标栏宽度约 64px，用于头像、任务、搜索、同步、提醒、帮助等入口。
- 清单侧栏宽度约 220px，承载快捷视图和用户清单。
- 任务列表弹性占满剩余空间，详情面板宽度约 420px。

## 视觉
- 主色使用低饱和青绿色，避免大面积高饱和色。
- 背景使用浅灰和浅青绿分层，列表区域保持白色或近白。
- 控件圆角控制在 4px 到 8px。
- 信息密度偏桌面工具型，避免营销式大卡片。
- 图标风格保持轻量线性或简洁符号；结构性入口避免使用平台差异大的 emoji。
- 所有可点击控件需要有 hover、focus-visible 和 disabled 状态，焦点状态不能被移除。

## 任务列表
- 每行包含完成框、标题、日期、优先级、标签、子任务/评论/附件状态。
- 置顶任务在默认排序下优先显示。
- 已完成任务默认可折叠。
- 鼠标悬停时显示更多操作。
- 今天视图中新建任务默认设置今天截止日期；最近 7 天、已完成、垃圾桶、搜索视图不制造含义不清的新建行为。

## 详情面板
- 顶部放完成状态、日期提醒、优先级、模式切换。
- 标题应可直接编辑。
- 描述和检查事项使用清晰分区。
- 底部显示所属清单和常用操作。
- 菜单和日期选择器需要限制在窗口内，避免贴边溢出。
- 删除、永久删除、清除已完成等危险操作需要确认；普通删除优先进入垃圾桶。

## 空状态与弹窗
- 空状态保持轻量，不使用大面积插画。
- 菜单和弹窗点击外部关闭。
- 弹窗位置要避免超出窗口边界。
- 不展示不可用的同步、协作、账号、帮助等入口；如果未来需要占位，必须用禁用态和明确说明。
