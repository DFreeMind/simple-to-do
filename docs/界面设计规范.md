# 界面设计规范

## 布局
- 桌面端采用四栏布局：窄图标栏、清单侧栏、任务列表、任务详情。
- 窄图标栏宽度约 64px，用于头像、任务、搜索、同步、提醒、帮助等入口。
- 清单侧栏宽度约 220px，承载快捷视图和用户清单。
- 任务列表弹性占满剩余空间，详情面板宽度约 420px。

## 视觉
- 主色使用低饱和青绿色，避免大面积高饱和色。
- 背景使用浅灰和浅青绿分层，列表区域保持白色或近白。
- 控件圆角控制在 4px 到 8px。
- 信息密度偏桌面工具型，避免营销式大卡片。

## 任务列表
- 每行包含完成框、标题、日期、优先级、标签、子任务/评论/附件状态。
- 置顶任务在默认排序下优先显示。
- 已完成任务默认可折叠。
- 鼠标悬停时显示更多操作。

## 详情面板
- 顶部放完成状态、日期提醒、优先级、模式切换。
- 标题应可直接编辑。
- 描述和检查事项使用清晰分区。
- 底部显示所属清单和常用操作。

## 空状态与弹窗
- 空状态保持轻量，不使用大面积插画。
- 菜单和弹窗点击外部关闭。
- 弹窗位置要避免超出窗口边界。
