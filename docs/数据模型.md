# 数据模型

## 作用与权威来源

本文件描述前端与 Rust 层交换的**应用状态契约**，用于新增功能和兼容旧数据。SQLite 的物理列、索引与附件目录以 [数据库设计](数据库设计.md) 和 `src-tauri/src/main.rs` 为准；本文件不重复记录一次性历史方案。

当前前端状态版本为 **v12**。应用保存时，Pinia 将状态组合为一个 payload，Rust 再拆分写入关系表。

## 根状态

```ts
type AppState = {
  schemaVersion: number
  groups: Group[]
  lists: List[]
  taskGroups: TaskGroup[]
  tasks: Task[]
  trash: Task[]
  listTrash: ListTrash[]
  viewOrders: Record<string, string[]>
  profile: Profile
  settings: Settings
  clock: Clock
}
```

`schemaVersion` 用于前端状态兼容；缺失字段由迁移器和 normalize 补默认值。`trash` 与 `listTrash` 是产品层回收站状态，不等于 SQLite 表的删除标记。

## 核心实体

| 实体 | 关键字段 | 约束 |
| --- | --- | --- |
| Group | `id`、`name`、`collapsed`、`sortOrder` | 侧栏清单分组 |
| List | `id`、`name`、`groupId`、`color`、`viewMode`、`sortOrder` | `viewMode` 为 `list` 或 `group` |
| TaskGroup | `id`、`listId`、`name`、`emoji`、`color`、`sortOrder`、`collapsed` | 必须属于一个现存清单 |
| Task | `id`、`title`、`listId`、`completed`、`deleted`、`dueDate`、`reminderAt`、`repeatRule`、`priority`、`tags`、`subtasks`、`attachments` | `taskGroupId` 非空时必须与任务同属一个清单 |
| Subtask | `id`、`title`、`completed`、`sortOrder` | 通过任务嵌套保存，按排序显示 |
| Attachment | `id`、`kind`、`mime`、`originalName`、`relativePath`、`sha256`、`sizeBytes` | 产品当前只创建 `image`；预览 URL 仅存在运行时 |
| Profile | `id`、`nickname`、`avatarRelativePath`、`avatarSha256` | 是当前设备资料，不是云账号 |
| Settings | 外观、布局、已完成显示、通知、音效、窗口关闭和启动提示偏好 | 新增字段必须给默认值 |
| Clock | `profiles`、`focusSettings`、`activeSession`、`pendingBreak`、`cycleFocusCount`、`history`、`rhythm` | 专注模式、休息节奏、未结束会话、历史和节律提醒 |
| FocusProfile | `id`、`name`、`durationSeconds`、`description`、`sortOrder` | `durationSeconds` 为 `null` 时表示自由计时 |
| FocusSession | `id`、`profileId`、`taskId`、`status`、`phase`、`durationSeconds`、`createdAt`、`startedAt`、`elapsedSeconds` | `phase` 为 `focus`、`short-break` 或 `long-break`；`status` 为 `running` 或 `paused` |
| PendingBreak | `phase`、`profileId`、`durationSeconds`、`createdAt` | 专注完成后等待用户开始或跳过的短/长休息，不计入运行会话 |
| FocusHistory | `id`、`profileId`、`taskId`、`phase`、`startedAt`、`finishedAt`、`elapsedSeconds`、`result`、`reward`、`note` | 完成专注按时长记录奖励：1–9 分钟蓝莓、10–24 分钟草莓、25–44 分钟番茄、45–89 分钟西瓜、90 分钟及以上南瓜；`result` 为 `completed`、`abandoned` 或 `interrupted`；历史 `sesame` 奖励兼容映射为蓝莓；不自动完成任务 |
| Rhythm | `pausedUntil`、`reminders` | 全局暂停状态和可同时运行的节律提醒集合 |
| RhythmReminder | `title`、`enabled`、`triggerType`、`intervalSeconds`、`time`、`weekdays`、`workStart`、`workEnd`、`quietStart`、`quietEnd`、`lastCompletedAt`、`snoozedUntil`、`skippedDate` | `triggerType` 为 `interval`、`fixed-time` 或 `active-duration`；最后一种只有原生活跃时长能力可用时才允许启用 |

## 任务字段语义

- `myDayDate`：任务加入哪一天的今日视图，格式为 `YYYY-MM-DD`。
- `dueDate`：带可选时间的截止日期；`reminderAt` 是独立提醒时刻，`reminderDisabled` 表示用户明确关闭提醒。
- `repeatRule`：当前支持 `daily`、`weekdays`、`weekly`、`monthly`、`yearly`。
- `descriptionHtml`：富文本备注 HTML。嵌入图片保存为应用附件的相对路径，而不是 base64。
- `deletedByListId`：清单被删除时，用于将任务与原清单回收站关联并支持一起恢复。
- `comments`、`editorMode`：仅为历史数据兼容保留，当前界面不提供评论或编辑模式切换。

## 设置的稳定字段

`theme`、`themeBackgrounds`、`density`、`sidebarCollapsed`、`detailOpen`、`detailWidth`、`startView`、`completedVisible`、`groupCompletedDisplayMode`、`groupCompletedVisibleByDefault`、`showCompletionDuration`、`trashRetentionDays`、`soundEnabled`、各分类音效开关、任务与节律提醒通知及声音开关、`focusCompletionNotificationsEnabled`、`focusCompletionSoundEnabled`、`windowCloseBehavior`、`dailyGuidanceEnabled`、`dailyGuidanceStyle`。两个专注完成字段对旧数据默认均为 `true`，只控制后台系统通知及其声音，不关闭应用内完成反馈。

新设置必须：提供默认值、在旧状态 normalize 时补齐、确认是否要持久化，并同步更新设置界面、应用内帮助和产品说明。

## 兼容与迁移规则

1. 前端状态迁移由 `src/utils/migrator.js` 完成；目前兼容 v1–v12 状态。迁移前会创建迁移备份。
2. SQLite 结构迁移由 Rust 管理，使用独立的 `databaseSchemaVersion`；不得以修改前端 `schemaVersion` 代替表结构迁移。
3. 读取到未来版本、字段关系不合法或数据库不可读时，停止自动保存并显示保护错误，不能落回空数据。
4. 新字段优先可空或有安全默认值；删除字段前必须提供向后读取策略。
5. 任务与清单删除先进入回收站。附件引用移除后不会立刻删除物理文件，而是通过用户触发的扫描与清理站处理。
