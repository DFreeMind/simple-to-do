# 数据模型

## Root
- `schemaVersion: number`，当前为 `2`；旧 JSON 缺失或为 v1 时按兼容逻辑补齐字段。
- `groups: Group[]`
- `lists: List[]`
- `tasks: Task[]`
- `trash: Task[]`
- `viewOrders: Record<string, string[]>`，普通任务视图的用户拖拽顺序；key 包括 `today`、`inbox`、`planned`、`important`、`list:<id>`、`search:<query>`。
- `settings: Settings`

## Group
- `id: string`
- `name: string`
- `collapsed: boolean`
- `sortOrder: number`，侧栏分组手动排序权重。

## List
- `id: string`
- `name: string`
- `groupId: string | null`
- `color: string`
- `isSystem: boolean`
- `sortOrder: number`，侧栏清单手动排序权重。

## Task
- `id: string`
- `title: string`
- `description: string`，兼容旧纯文本描述。
- `descriptionHtml: string`，富文本备注 HTML。
- `completed: boolean`
- `completedAt: string | null`
- `deleted: boolean`
- `deletedAt: string | null`
- `pinned: boolean`
- `important: boolean`
- `myDayDate: string | null`，格式为 `YYYY-MM-DD`，表示加入某天的今日聚焦。
- `listId: string`
- `dueDate: string | null`
- `reminderAt: string | null`
- `repeatRule: "daily" | "weekly" | "monthly" | "yearly" | string | null`
- `priority: 0 | 1 | 2 | 3`
- `tags: string[]`
- `subtasks: Subtask[]`
- `attachments: Attachment[]`
- `comments: Comment[]`，仅保留旧数据兼容，当前 UI 不展示评论入口。
- `editorMode: string`，兼容旧模式字段，当前 UI 不再使用模式切换。
- `createdAt: string`
- `updatedAt: string`

## Subtask
- `id: string`
- `title: string`
- `completed: boolean`
- `sortOrder: number`，子任务手动排序权重。

## Attachment
- `id: string`
- `path: string`
- `url: string`
- `createdAt: string`

## Settings
- `theme: "mint" | "blue" | "violet" | "graphite"`
- `density: "comfortable" | "compact"`
- `detailOpen: boolean`
- `startView: "today" | "inbox" | "planned" | "important"`
- `completedVisible: boolean`

## 兼容规则
- 旧 `groups`、`lists` 缺少 `sortOrder` 时按现有数组顺序生成默认排序。
- 旧 `lists` 缺少 `groupId`、`color` 时使用默认分组和颜色补齐。
- 旧 `tasks` 缺少 `important`、`myDayDate`、`attachments` 等字段时使用默认值。
- 旧 `subtasks` 缺少 `sortOrder` 时按原数组顺序生成默认排序。
- 旧数据缺少 `viewOrders` 时保持当前智能排序；用户首次拖拽后才写入对应视图顺序。
- 旧 `comments` 和 `editorMode` 保留在数据中，避免迁移丢失。
- 删除任务先复制到 `trash`，永久删除才从 `tasks` 和 `trash` 移除。
