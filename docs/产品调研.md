# 产品调研

## 调研结论

主流待办产品已经覆盖任务、日历、习惯、番茄、协作和跨设备同步。易简清单不以“功能更全”为竞争方式，而以**本地优先、桌面上的复杂事项推进、克制且可恢复的数据整理**为取舍标准。

时钟模块新增后，产品的合理边界是“任务推进过程中的专注与休息节奏”，不是扩张为习惯追踪、健康诊断或考勤产品。番茄钟解决开始和结束一段专注；久坐提醒解决连续使用电脑过久。两者共用通知与后台计时能力，但触发逻辑、记录和用户预期应保持分开。

调研结论用于判断方向，不等于功能承诺；当前已交付能力以 [产品需求文档](产品需求文档.md) 为准。

## 时钟与休息提醒专项调研（2026-07）

| 产品/类别 | 核心做法 | 对易简清单的启发 | 不建议照搬 |
| --- | --- | --- | --- |
| TickTick 番茄钟 | 从任务进入专注周期；支持自定义专注/休息时长、估算番茄数、白噪声与记录 | 把“本次专注什么”放在启动动作里，完成后留下轻量记录 | 用日历、习惯、统计等大量模块包裹计时功能 |
| Pomofocus 等极简番茄钟 | 默认 25/5 节奏、清晰倒计时、阶段自动切换，任务仅作轻量上下文 | 首屏只需显示当前阶段、剩余时间和一个明确动作；预设必须可改 | 以连续循环和声音强迫用户进入固定节奏 |
| Stretchly | 短暂微休息与较长休息分层，均可独立配置 | “抬眼/活动一下”和“离开座位”应有不同默认时长和文案 | 将复杂参数直接暴露在主界面 |
| Workrave | 基于键鼠活跃时间累计；空闲时间可算作自然休息；到点先温和提示，再允许延后或跳过 | 久坐提醒应优先按活跃时长，而非只按墙上时钟；要识别系统锁屏、睡眠与用户离开 | 强制阻塞键鼠、每日电脑使用上限、过度详细的合规统计 |
| Time Out（macOS） | 微休息/常规休息、可延后/跳过，能按会议或前台应用等规则避免打扰 | 后续可考虑“全屏/会议中不打扰”的平台适配，先保留设置入口的扩展空间 | 依赖第三方应用识别和脚本规则作为 MVP 前提 |

### 可复用的交互规律

1. **先温和、后明确**：提醒先以系统通知或小浮层提示；用户可“开始休息 / 延后 5 分钟 / 本次跳过”。不要在首次版本遮挡工作窗口或锁定输入。
2. **让自然离席生效**：检测到连续空闲达到休息时长，即记为已休息且重置下一轮；从睡眠、锁屏恢复后不补发过期提醒。
3. **把任务关联设为可选**：启动专注前可选一个任务或保持“无任务专注”；计时不应要求用户先整理任务。
4. **记录只服务回顾**：保留每次专注的开始、结束、阶段和关联任务；首页仅显示今日专注次数/时长、休息完成次数。不要在 MVP 做排行、连续打卡或健康评分。
5. **默认值是建议，不是处方**：提供“番茄 25/5、4 轮后长休息 15 分钟”和“微休息 30 秒/30 分钟、长休息 5 分钟/60 分钟”等可修改预设，并说明应按个人情况调整；不作医疗或人体工学效果承诺。

## 产品取舍

### 必须持续做好

- 本机即可完成记录、安排、拆分、提醒、完成和恢复。
- 今日、计划、收集箱和任务详情之间的切换足够快，任务属性不会遮蔽下一步。
- 删除、附件清理和数据恢复有可理解的回退路径。
- Windows 与 macOS 都提供贴合桌面的窗口、通知和安装体验。

### 需要证明确有价值才做

- 周视图、轻量月历、统计、导出、批量整理和标签管理。
- 专注白噪声、预设库、每周趋势、全屏/会议自动免打扰、活动强度阈值与跨设备统计。
- 这些能力只有在帮助用户拆清或持续推进复杂事项时才进入路线图，不能因为竞品具备就加入。

### 当前不纳入

- 账号、云同步、共享清单、团队协作、评论与权限。
- 习惯、四象限、泛化项目管理、健康诊断、强制锁屏休息、摄像头/姿态检测、系统级应用使用审计。
