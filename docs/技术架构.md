# 技术架构

## 总体架构
- 内嵌前端：Vue 3 + Pinia + Vite + SCSS，仅作为 Tauri GUI 的渲染层。
- 桌面运行时：Tauri 2。
- 原生层：Rust command 负责本地文件和系统能力。
- 数据存储：应用数据目录中的 SQLite 数据库 `simpletodo.db`。
- 附件存储：图片由 Rust 层导入到应用数据目录的分层 `attachments/` 文件夹，业务数据只保存元数据和相对路径。

## 前端职责
- 渲染主界面、任务列表、详情面板、弹窗和搜索。
- 使用 Pinia 管理任务、清单、分组、设置状态和派生视图。
- 通过 `src/services/platform.js` 调用平台能力，不直接依赖 Tauri 或 Electron 的全局对象。
- 不维护独立 Web 版本；开发、验证和交付都以 Tauri 桌面窗口为准。
- 结构性图标统一使用 `lucide-vue-next`，富文本备注使用 Tiptap。

## Rust/Tauri 职责
- 提供 `load_data` 和 `save_data` 命令，在前端状态结构和 SQLite 表之间转换。
- 提供 `select_image` 命令选择图片文件。
- 提供 `import_image` 命令将图片复制到分层附件目录，并返回 hash、MIME、尺寸和相对路径等元数据。
- 提供 `read_attachment` 命令按相对路径读取附件预览。
- 提供 `cleanup_orphan_attachments` 命令清理未被业务数据引用的附件文件。
- 控制窗口尺寸、标题和安全策略。

## 命令接口
- `load_data() -> Option<serde_json::Value>`
- `save_data(data: serde_json::Value) -> bool`
- `select_image() -> Option<String>`
- `read_image(file_path: String) -> Option<String>`
- `import_image(file_path: String) -> Attachment`
- `read_attachment(relative_path: String) -> Option<String>`
- `cleanup_orphan_attachments(data: serde_json::Value) -> u32`

## 数据兼容
- 当前仍处于开发阶段，不迁移旧 `todo-data.json`。
- 前端状态结构继续由 Pinia normalize 补齐缺失字段。
- Rust 层负责 SQLite schema 初始化、事务写入和从表结构组装前端状态。
- Rust 层保存附件元数据前会移除外置附件的预览 `url`，避免图片 base64 写入数据库。
- SQLite 表设计、附件目录和清理策略见 `docs/数据库设计.md`。

## 开发态数据位置
- `npm run dev` 等同于 `tauri dev`，会启动 Tauri 桌面窗口并调用 Rust command；数据写入 Tauri `app_data_dir()`，不是浏览器 localStorage。
- 当前应用标识为 `cn.duqimeng.simpletodo`。Windows 下开发态主数据路径为 `%APPDATA%\cn.duqimeng.simpletodo\simpletodo.db`，附件目录为 `%APPDATA%\cn.duqimeng.simpletodo\attachments\`。
- 由于开发态和安装包使用同一个应用标识，默认会读写同一份用户数据。需要隔离测试数据时，应临时改用独立 identifier 或在启动前备份/清理该目录。
- 单独运行 `npm run frontend:dev` 只启动 Vite 浏览器页面；此时没有 Tauri 环境，平台服务会退回到浏览器 `localStorage`，key 为 `simple-to-do:data`，且不能使用本地图片导入 command。

## Windows 打包
- Windows 安装包使用 Tauri NSIS target，配置位于 `src-tauri/tauri.conf.json` 的 `bundle.windows.nsis`。
- NSIS 安装器语言固定为 `SimpChinese`，不显示语言选择器，安装和卸载流程应全部使用中文文案。
- 安装器图形资源放在 `src-tauri/nsis/`：`header.bmp` 为 493x312，`sidebar.bmp` 为 164x314。
- `installer-art-source.png` 是安装器位图的源插画，后续调整安装器视觉时先更新源图，再导出对应 BMP。

## 主要模块
- `src/stores/task.js`：任务、清单、分组、设置、搜索、计划分组、月历派生、统计派生和本地保存。
- `src/components/Sidebar.vue`：系统视图、搜索、清单分组和设置入口。
- `src/components/TaskList.vue`：视图标题、快速添加、今日建议、计划分组和任务列表。
- `src/components/CalendarView.vue`：月历任务展示、选中日期日程侧栏、按日期新建任务和拖拽改期。
- `src/components/YearMonthPicker.vue`：复用的年月选择浮层，用于月历和任务详情日期选择器。
- `src/components/StatsView.vue`：个人复盘统计卡片、7 天趋势和清单分布。
- `src/components/TaskItem.vue`：任务行展示和行内操作。
- `src/components/TaskDetail.vue`：统一详情面板、属性编辑、子任务、富文本备注和附件。
- `src/components/SettingsPanel.vue`：主题、密度、启动页和界面偏好。
