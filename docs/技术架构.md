# 技术架构

## 总体架构
- 内嵌前端：Vue 3 + Pinia + Vite + SCSS，仅作为 Tauri GUI 的渲染层。
- 桌面运行时：Tauri 2。
- 原生层：Rust command 负责本地文件和系统能力。
- 数据存储：应用数据目录中的 SQLite 数据库 `simpletodo.db`。
- 本机身份：SQLite 保存一个独立的本地 Profile（昵称和头像元数据）；它不依赖账号系统，未来可与云端身份绑定。
- 附件存储：图片由 Rust 层导入到应用数据目录的分层 `attachments/` 文件夹，业务数据只保存元数据和相对路径。
- 系统提醒：通过 Tauri 2 notification 插件发送即时系统通知；Pinia 维护单一桌面端提醒调度器，在应用运行、从系统休眠恢复或应用重启后投递到期提醒，并以任务中的投递状态去重。
- 应用更新：使用 `tauri-plugin-updater` 从 GitHub Release 的 `latest.json` 获取版本信息，并仅安装由内置公钥验证通过的更新包。
- 数据升级保护：Rust 层以 `meta.databaseSchemaVersion` 记录 SQLite 结构版本，并与前端业务 `schemaVersion` 分离。仅在结构版本落后、确有 migration plan 时，才使用 SQLite Online Backup API 创建一致性备份并在单个事务中升级；数据库无法读取时，前端必须阻断主界面和全部写入操作，不得以默认空状态替代用户数据。
- 用户恢复点：Rust command `create_data_backup`、`list_data_backups`、`data_backup_location`、`open_data_backup_location`、`open_data_backup`、`delete_data_backup`、`restore_data_backup` 统一管理 `backups/` 下的快照。快照使用 SQLite Online Backup API 复制数据库，并一并复制 `attachments/`、`profile/`；恢复采用暂存目录和逐项回滚，开始前强制创建“恢复前安全点”。`open_data_backup` 在 Windows 资源管理器/Mac Finder 中定位单项快照；删除命令仅接受经过格式和元数据校验的快照 ID。

## 前端职责
- 渲染主界面、任务列表、详情面板、弹窗和搜索。
- 使用 Pinia 管理任务、清单、分组、设置状态和派生视图。
- 使用 Pinia 管理个人资料的展示与编辑状态；当前资料随任务状态一同由 `task` store 保存，账号和协作能力落地前再拆出独立 store。头像选择、导入和读取仍统一经 `src/services/platform.js`，不由组件直接读写本地路径。
- 通过 `src/services/platform.js` 调用平台能力，不直接依赖 Tauri 或 Electron 的全局对象。
- 不维护独立 Web 版本；开发、验证和交付都以 Tauri 桌面窗口为准。
- 结构性图标统一使用 `lucide-vue-next`，富文本备注使用 Tiptap。
- `src/services/platform.js` 只封装通知权限与即时投递；Pinia 根据任务生命周期维护下一条提醒的定时器，修改截止日期时会保留提醒相对截止时间的偏移，避免日期移动后提醒留在旧日期。

## Rust/Tauri 职责
- 提供 `load_data` 和 `save_data` 命令，在前端状态结构和 SQLite 表之间转换。
- 提供 `select_image` 命令选择图片文件。
- 提供 `import_image` 命令将图片复制到分层附件目录，并返回 hash、MIME、尺寸和相对路径等元数据。
- 提供 `read_attachment` 命令按相对路径读取附件预览。
- 提供存储健康扫描和清理站命令：扫描未引用附件、移入清理站、恢复或永久删除；不在业务删除流程中静默删除附件。
- 注册 `tauri-plugin-notification`，使用 `src-tauri/capabilities/default.json` 开放 notification 默认权限。
- 注册 `tauri-plugin-updater`，使用 `src-tauri/capabilities/desktop.json` 开放 updater 默认权限；私钥仅保存在 GitHub Secret `TAURI_SIGNING_PRIVATE_KEY`，严禁提交到仓库。
- 管理系统托盘：关闭窗口时最小化到托盘（`on_window_event` 拦截 `CloseRequested`），托盘菜单支持显示/退出。
- 控制窗口尺寸、标题和安全策略。

## 身份与同步演进
- 侧栏顶部固定为个人空间入口：当前显示本地头像、昵称和“个人空间 · 本地数据”状态；未来可在原位置显示登录账户、同步状态和空间切换器。
- 个人资料与任务数据分离。`profile.id` 是本机稳定 UUID，不等同于未来服务端用户 ID；账号绑定后通过可空的 `account_id` 建立映射，不能以服务端 ID 覆盖本机 ID。
- 头像文件使用应用数据目录内的专用 `profile/avatars/` 内容寻址路径，数据库只保存相对路径和 hash。替换或恢复默认头像时，仅在没有引用后通过存储维护流程清理旧文件。
- 云同步以本地 SQLite 为可读写副本；同步元数据、设备标识和冲突状态应使用独立字段/表，不能改变离线可用和本地资料可编辑的行为。
- 个人空间 UI 作为身份与数据服务的唯一聚合入口，采用左侧分类 + 右侧内容的空间中心布局，固定分区为身份资料、空间、连接与协作、数据与安全。当前未上线的分区只显示状态，不暴露不可用的交互控件。

## 命令接口
- `load_data() -> Option<serde_json::Value>`
- `save_data(data: serde_json::Value) -> bool`
- `select_image() -> Option<String>`
- `read_image(file_path: String) -> Option<String>`
- `import_image(file_path: String) -> Attachment`
- `read_attachment(relative_path: String) -> Option<String>`
- `scan_storage_health() -> StorageHealth`
- `quarantine_orphan_attachments(relativePaths: string[]) -> StorageOperation`
- `restore_quarantined_attachments(itemIds: string[]) -> StorageOperation`
- `purge_quarantined_attachments(itemIds: string[]) -> StorageOperation`

## 数据兼容
- 当前仍处于开发阶段，不迁移旧 `todo-data.json`。
- 运行时 SQLite 数据迁移必须先备份再执行。每次新增结构变更都要递增 `DATABASE_SCHEMA_VERSION`、新增对应 migration plan 和测试；无结构变更的应用升级只执行只读 `quick_check`，不备份、不迁移、不改写数据库。当前标识目录没有数据库时，应用会尝试从历史 Tauri 标识 `com.simpletodo.desktop`、`com.simpletodo.app` 导入数据库以及附件/头像目录；源目录保留不动。
- 数据库读取、备份或迁移失败时，应用进入只允许重试读取的保护页；不会显示任务主界面，也不会写入默认数据。此规则同时适用于手动安装和应用内更新。
- 前端状态结构继续由 Pinia normalize 补齐缺失字段。
- Rust 层负责 SQLite schema 初始化、事务写入和从表结构组装前端状态。
- Rust 层保存附件元数据前会移除外置附件的预览 `url`，避免图片 base64 写入数据库。
- SQLite 表设计、附件目录和清理策略见 `docs/数据库设计.md`。

## 开发态数据位置
- `npm run dev` 等同于 `tauri dev`，会启动 Tauri 桌面窗口并调用 Rust command；数据写入 Tauri `app_data_dir()`，不是浏览器 localStorage。
- 当前应用标识为 `cn.duqimeng.simpletodo`。Windows 下开发态主数据路径为 `%APPDATA%\cn.duqimeng.simpletodo\simpletodo.db`，附件目录为 `%APPDATA%\cn.duqimeng.simpletodo\attachments\`。
- 由于开发态和安装包使用同一个应用标识，默认会读写同一份用户数据。需要隔离测试数据时，应临时改用独立 identifier 或在启动前备份/清理该目录。
- 单独运行 `npm run frontend:dev` 只启动 Vite 浏览器页面；此时没有 Tauri 环境，平台服务会退回到浏览器 `localStorage`，key 为 `simple-to-do:data`，且不能使用本地图片导入 command。

## Windows 打包
- Windows 安装包使用 Tauri NSIS target，配置位于 `src-tauri/tauri.conf.json` 的 `bundle.windows.nsis`。
- NSIS 安装器语言固定为 `SimpChinese`，不显示语言选择器，安装和卸载流程应全部使用中文文案。
- 安装器图形资源放在 `src-tauri/nsis/`：`header.bmp` 为 493x312，`sidebar.bmp` 为 164x314。
- `installer-art-source.png` 是安装器位图的源插画，后续调整安装器视觉时先更新源图，再导出对应 BMP。
- `bundle.createUpdaterArtifacts` 会额外生成 NSIS 安装包的 `.sig` 签名文件；正式发布时由 `.github/workflows/release.yml` 生成并上传 `latest.json`，更新器端点固定为 GitHub 最新 Release。
- 发布工作流仅支持手动触发，必须输入中文 Release Notes；它先创建草稿 Release、构建 Windows NSIS 资源并生成更新清单。发布人核验版本、资源名和说明后再公开 Release。

## 主要模块
- `src/stores/task.js`：任务、清单、任务分组、侧栏分组、个人资料、设置、搜索、排序、视图切换和本地保存；账号与协作能力落地前应拆出 `profile` store，避免身份资料与任务业务继续耦合。
- `src/components/Sidebar.vue`：系统视图、搜索、清单分组、设置入口和个人空间入口；折叠态渲染 56px Rail 图标栏，保留个人头像按钮与清单浮动卡片。
- `src/components/ProfilePanel.vue`：个人资料弹窗，支持修改昵称、更换或恢复默认头像，并在“数据与安全”管理本机恢复点。
- `scripts/release-preflight.mjs`：发布前检查应用版本一致性和 Windows Release 工作流的 tag、草稿、NSIS、签名更新资源命名约束；工作流必须传入本次版本才能开始构建。
- `src/components/TaskList.vue`：视图标题、快速添加、今日建议、计划分组、任务分组视图和任务列表；在分组模式中按未完成/已完成桶渲染任务，并由各分组头提供临时显示已完成任务的开关。
- `src/components/TaskGroupHeader.vue`：任务分组头部，支持图标、颜色、折叠、重命名、删除操作。
- `src/components/TaskItem.vue`：任务行展示和行内操作。
- `src/components/TaskDetail.vue`：统一详情面板、属性编辑、子任务与富文本备注。
- `src/components/ViewModeToggle.vue`：清单视图模式切换（列表/分组）。
- `src/components/EmojiPicker.vue`：emoji 选择器，用于任务分组图标。
- `src/components/SettingsPanel.vue`：主题配色、主题背景开关、密度、侧栏折叠、启动视图、已完成任务的列表/分组展示规则、垃圾桶保留天数、操作音效（任务/清单/分组独立开关）和关于信息；“关于”页提供手动检查、下载和安装已签名更新。
- `src/utils/dailyMessages.js`：本地每日提示文案库与确定性选择器。以本地日期、场景和提示风格生成稳定文案；不读取网络、不传输任务内容，同一日期不因刷新变化。
- `src/components/HelpCenter.vue`：应用内帮助中心，以文档元数据（id、分类、标题、摘要、关键词正文、章节）渲染快速开始、手册、常见说明和更新内容；后续新增文档应扩展该内容集合，保持单一阅读入口。
- `src/services/platform.js`：封装 Tauri command、附件读取和系统提醒通知投递；非 Tauri 环境降级为不可用。
- `src/utils/sound.js`：基于 HTML Audio 的短促 UI 音效播放层，按任务、清单、分组分类开关并对拖拽高频提示节流；素材位于 `src/assets/sounds/`，仅使用 CC0 可再分发资源并随附来源与许可证说明。
- `src/utils/migrator.js`：数据版本迁移器，支持跨版本升级；当前最新版本为 v8，补齐任务级提醒关闭状态，并将截止时间作为默认提醒时间。
- `src/components/CalendarView.vue`：月历视图（已实现但未接入侧栏入口）。
- `src/components/StatsView.vue`：统计视图（已实现但未接入侧栏入口）。
