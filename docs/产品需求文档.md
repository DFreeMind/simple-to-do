# 产品需求文档

## 目标
构建一款本地优先、跨平台的桌面待办应用，第一阶段复刻滴答清单的核心任务体验，并为后续日历、四象限、番茄和习惯模块预留结构。

## 目标用户
- 需要在桌面端高频记录和整理任务的个人用户。
- 需要清单、日期、优先级、标签和搜索来管理工作与生活任务的用户。
- 希望应用轻量、响应快、数据本地可控的用户。

## 核心场景
- 快速把临时事项放入收集箱。
- 按今天、最近 7 天、清单查看待办。
- 在任务详情中维护描述、检查事项、附件、评论和任务属性。
- 通过搜索、标签和优先级快速定位任务。
- 删除任务后可从垃圾桶恢复或永久删除。

## MVP 范围
- 本地任务数据持久化。
- 清单管理：新增、重命名、删除。
- 任务管理：新增、编辑、完成、置顶、复制、移动、删除、恢复。
- 任务属性：截止日期、提醒时间、优先级、标签、子任务、附件、评论。
- 视图：今天、最近 7 天、收集箱、清单、已完成、垃圾桶、搜索。

## 后续范围
- 日历视图。
- 四象限视图。
- 番茄专注。
- 习惯打卡。
- 账号、同步和协作。
