# 产品需求文档

## 目标
构建一款本地优先、跨平台的桌面待办应用「易简清单」。第一阶段聚焦本地个人任务管理，把收集、整理、查看、编辑、完成、删除和恢复做到稳定可日用，并以滴答清单的核心桌面体验作为布局与交互参考。

## 目标用户
- 需要在桌面端高频记录和整理任务的个人用户。
- 需要清单、日期、优先级、标签和搜索来管理工作与生活任务的用户。
- 希望应用轻量、响应快、数据本地可控的用户。

## 核心场景
- 快速把临时事项放入收集箱。
- 按今天、最近 7 天、清单查看待办。
- 在任务详情中维护描述、检查事项、附件、评论和任务属性。
- 通过搜索、标签和优先级快速定位任务。
- 删除任务后可从垃圾桶恢复或永久删除。

## 可日用 MVP 范围
- 本地任务数据持久化。
- 清单管理：新增、重命名、删除。
- 任务管理：新增、编辑、完成、置顶、复制、移动、删除、恢复。
- 任务属性：截止日期、提醒时间、优先级、标签、子任务、附件、评论。
- 视图：今天、最近 7 天、收集箱、清单、已完成、垃圾桶、搜索。

## 本轮打磨标准
- 所有可见入口必须对应真实可用功能；同步、账号、协作、帮助中心等暂不展示为可点击入口。
- 今天、最近 7 天、清单、搜索、已完成、垃圾桶视图的新建、筛选、删除和恢复行为要清晰一致。
- 数据保存失败不能静默丢失，需要给出可见错误提示并保留用户当前编辑状态。
- 视觉保持低饱和青绿色、紧凑信息密度、三栏/四栏桌面布局，避免营销式空状态和大卡片。
- 菜单、弹窗和危险操作要有明确关闭路径、确认或恢复路径。

## 后续范围
- 日历视图。
- 四象限视图。
- 番茄专注。
- 习惯打卡。
- 账号、同步和协作。
