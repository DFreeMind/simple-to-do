# 易简清单文档中心

这里是易简清单（Simple To Do）的项目文档入口。文档按读者和用途分层：先帮助使用者理解和安装产品，再说明当前产品边界，最后提供开发与数据维护所需的技术事实。

| 产品解决的问题、已交付和明确不做什么 | [产品需求文档](产品需求文档.md) | 产品、设计、贡献者 |

## 文档状态约定

- **当前能力**：已在应用中提供，说明必须能在当前代码和界面中验证。
- **规划**：方向已经确定，但尚未作为可用入口交付；不会写进安装说明或功能清单。
- **历史记录**：保留迁移原因或旧方案，不代表当前实现。历史 Electron 迁移记录见 [Electron 迁移 Tauri 方案](Electron迁移Tauri方案.md)，任务分组的早期设计记录见 [根目录 plan.md](../plan.md)。

维护文档时，先确认代码、Release 资产和应用内帮助中心的真实状态；产品决策写入产品需求，平台接口写入技术架构，字段与持久化结构分别写入数据模型和数据库设计，避免同一事实在多处漂移。
