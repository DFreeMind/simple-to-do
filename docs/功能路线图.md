# 功能路线图

## 已完成的产品基础

- 本地个人任务管理：收集箱、今日、计划、重要、搜索、已完成和垃圾桶。
- 清单与任务分组、拖拽排序、任务详情、子任务、富文本备注和图片附件。
- 截止日期、提醒、重复任务、优先级、标签、快速添加解析和逾期重排。
- 个人空间、本机头像、附件清理站、垃圾桶保留期和本机恢复点。
- Windows 与 macOS 的桌面窗口、通知、后台恢复；Web Audio 操作反馈。
- SQLite 主存储、旧状态兼容、发布前校验和 GitHub Release 交付链路。

## 近期方向：交付完整的专注与节律产品

时钟不是以 MVP 名义无限延后的试验项。下面的顺序是工程依赖与可验证切片，目标是完成 [产品需求文档](产品需求文档.md) 所定义的完整首版产品；每个切片完成后都应可独立稳定使用。

1. **时钟核心与持久化基础**：实现统一计时状态机、专注模式、个人预设、任务关联、专注结果和数据库迁移；先保证后台、重启、睡眠恢复后的计时正确。
2. **节律提醒引擎**：支持按间隔、连续活跃时长与固定时刻触发的多条提醒，并实现模板、自定义、工作时段、静默、完成、延后、跳过与通知控制。
3. **原生桌面能力**：补充 Windows/macOS 的空闲时长检测、自然休息、托盘/菜单栏快捷控制、通知权限、睡眠/锁屏恢复及明确的平台降级。
4. **完整工作台体验**：完成主时钟、迷你悬浮计时器、休息界面、首次引导、设置页、键盘操作与无障碍状态表达；统一通知、声音和视觉提示。
5. **回顾、数据控制与帮助**：完成日/周/月回顾、任务与提醒维度的记录、删除/保留控制、隐私说明、测试通知与应用内使用指南。
6. **全场景质量验证**：验证多显示器、全屏/演示、系统勿扰、最小化后台、异常退出、时钟漂移、通知权限拒绝与 Windows/macOS 行为；无法等价实现的能力提供可理解的替代结果。
7. **任务推进质量**：继续打磨子任务、任务分组、计划与逾期重排，使下一步更清楚、变更后更容易恢复节奏。
8. **整理与回顾效率**：评估标签管理、批量整理、快速添加解析增强等能力，只在能减少维护成本时纳入。
9. **可理解的产品展示**：保持应用内帮助、截图、README、安装说明和 Release Notes 与实际版本一致。

## 需要先验证再决定的方向

- 周视图、轻量月历、统计或导出：只有在能帮助用户安排或复盘复杂事项时再进入计划，不能仅因竞品具备而加入。
- 跨设备同步、账号、团队共享、复杂自动化、摄像头/姿态识别和医疗化健康分析：这些会改变本地优先和隐私边界，不纳入当前完整产品。
- 账号、云同步、设备管理和协作：必须保留本地离线可用、数据可恢复和个人资料不依赖登录的前提；当前不排期。

## 当前不排期

- 习惯、四象限。
- 团队评论、成员权限、共享空间。
- 外部日历同步和泛化的项目管理能力。
- 健康诊断、强制锁屏休息、日常行为打卡与公开排行。

每一项进入开发前，需要写清楚：解决的真实任务推进问题、最小可交付范围、Windows/macOS 影响、数据兼容方案，以及 README/帮助中心/Release Notes 是否需要同步更新。
