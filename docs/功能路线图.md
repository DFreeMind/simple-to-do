# 功能路线图

## 已完成的产品基础

- 本地个人任务管理：收集箱、今日、计划、重要、搜索、已完成和垃圾桶。
- 清单与任务分组、拖拽排序、任务详情、子任务、富文本备注和图片附件。
- 截止日期、提醒、重复任务、优先级、标签、快速添加解析和逾期重排。
- 个人空间、本机头像、附件清理站、垃圾桶保留期和本机恢复点。
- Windows 与 macOS 的桌面窗口、通知、后台恢复；Web Audio 操作反馈。
- SQLite 主存储、旧状态兼容、发布前校验和 GitHub Release 交付链路。

## 近期方向：让复杂事项推进得更稳

按价值排序，而非按“功能数量”排序：

1. **时钟 MVP：专注周期**：实现可暂停、可恢复且可关联任务的专注/短休息/长休息状态机；先保证后台、重启、睡眠恢复后的计时与通知正确，再做视觉装饰。
2. **时钟 MVP：节律提醒引擎**：支持按间隔、连续活跃时长与固定时刻触发的多条提醒，并统一实现工作时段、静默、完成、延后、跳过与通知控制；护眼、补水、站立/活动和久坐均是可编辑预设，不分别开发为孤立功能。
3. **活跃时长与自然休息**：补充 Windows/macOS 原生空闲时长能力，落实自然休息、睡眠/锁屏恢复和平台降级；无法可靠检测的平台显式降级，不做输入阻塞。
4. **时钟回顾与隐私**：在本机持久化必要的专注和提醒响应记录，提供今天摘要、记录删除与清晰的隐私说明；不做排行榜、连续打卡和跨设备统计。
5. **任务推进质量**：继续打磨子任务、任务分组、计划与逾期重排，使下一步更清楚、变更后更容易恢复节奏。
6. **整理与回顾效率**：评估标签管理、批量整理、快速添加解析增强等能力，只在能减少维护成本时纳入。
7. **可靠性与跨平台体验**：持续验证 Windows/macOS 的通知、窗口恢复、安装更新和数据恢复路径，优先修复会影响日用的差异。
8. **可理解的产品展示**：保持应用内帮助、截图、README、安装说明和 Release Notes 与实际版本一致。

## 需要先验证再决定的方向

- 周视图、轻量月历、统计或导出：只有在能帮助用户安排或复盘复杂事项时再进入计划，不能仅因竞品具备而加入。
- 白噪声、更多提醒预设、周趋势、会议/全屏免打扰、每日电脑使用上限：在基础计时和提醒确有日用价值后，再依据隐私、平台能力和维护成本决定。
- 账号、云同步、设备管理和协作：必须保留本地离线可用、数据可恢复和个人资料不依赖登录的前提；当前不排期。

## 当前不排期

- 习惯、四象限。
- 团队评论、成员权限、共享空间。
- 外部日历同步和泛化的项目管理能力。

每一项进入开发前，需要写清楚：解决的真实任务推进问题、最小可交付范围、Windows/macOS 影响、数据兼容方案，以及 README/帮助中心/Release Notes 是否需要同步更新。
